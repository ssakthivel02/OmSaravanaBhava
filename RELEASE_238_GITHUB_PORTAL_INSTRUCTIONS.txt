WHY THIS RELEASE HAS TWO STAGES
GitHub browser upload can add and replace files but cannot remove tracked files
that are absent from the ZIP. Releases 235, 236 and 237 therefore left thirteen
compiled Python cache files and the retired site-directory.js file tracked.

STAGE 1 — NORMAL GITHUB BROWSER UPLOAD
1. Extract OmSaravanaBhava-Release-238.zip.
2. Upload the extracted contents to the repository root.
3. Preserve the .github directory.
4. Use the exact bootstrap title shown above.
5. Commit to main.

STAGE 2 — AUTOMATIC
The Release 238 Finalizer workflow starts after the bootstrap push. It:
- verifies the bootstrap parent and exact title;
- removes all fourteen planned paths with git rm;
- upgrades governance metadata to the actual bootstrap SHA;
- generates RELEASE_238_FINALIZATION_RESULT.json;
- runs deployment, hygiene, integrity and strict release gates;
- creates the exact final Release 238 commit;
- pushes the final commit to main.
