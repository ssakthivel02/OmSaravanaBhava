Reading and evidence boundary

  • No devotional text, selected passage or page body is copied into desktop-navigation state.
  • Route selection is temporary UI state and creates no new browser-storage key.
  • Progress is an approximate scroll position, not proof that a work has been read or completed.
  • Each destination retains its existing publication, source-review, completeness and copyright statements.
  • Saving a route uses the existing bounded reading-list dataset only after an explicit user action.
  • No account, analytics, behavioural profiling or cross-device synchronization is used.
