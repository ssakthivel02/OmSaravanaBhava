Safe quick fixes

Use the smallest recovery action first.

Old homepage or old styles

Reload once. When an old service-worker cache persists, clear only runtime/data caches before removing the full site storage.

Page did not refresh

Reload the current route after cache clearing. A hard refresh may be required when a proxy or CDN retains an earlier asset.

Missing or renamed route

The 404 page checks an exact alias registry, then suggests related governed routes without fuzzy automatic redirects.

Open directory

Offline page unavailable

Open the site once online so core routes and assets can enter the PWA cache. Optional routes may still require a network connection.

Open offline guide

Notes or favourites missing

Browser-local data is tied to the browser profile and site storage. Restore a prior export where available.

Personal data tools

Content looks incorrect

Do not overwrite a source-controlled text from memory. Prepare a correction with page, edition and evidence.

Prepare correction

Escalation information to include

Page URL, approximate time, browser/device, online/offline state, visible error, expected result, steps to reproduce and whether a cache refresh changed the behaviour. Never include passwords, private notes or unnecessary personal data.
