Data and editorial boundary

  • Collection storage contains only collection identifiers, names, descriptions, route references and timestamps.
  • Devotional text, selected passages, page-body content, images and audio files are not copied into collections.
  • Route titles and publication metadata always come from the current route directory.
  • Removed or no-longer-eligible routes are reported as stale references and can be removed explicitly.
  • Collections are included in Release 227 Personal Data backup and restore as the sixth registered local dataset.
  • No account, cloud synchronization, analytics, recommendation engine or collaborative service is used.
