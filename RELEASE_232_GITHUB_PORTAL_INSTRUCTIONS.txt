WHY THIS RELEASE IS SMALLER THAN 2,000 FILES
Release 232 intentionally rejects file-count-driven delivery. It contains 60 purposeful files. A 2,000-file browser upload would be difficult to inspect, easy to misplace and unsafe to roll back. The new gate warns above 100 files and fails above 500.

DOWNLOAD AND EXTRACT
1. Download OmSaravanaBhava-Release-232.zip.
2. Verify the ZIP SHA256 supplied with the package.
3. Extract the ZIP completely.
4. In Windows File Explorer enable View > Show > Hidden items.
5. Confirm the hidden .github directory is visible.
6. Do not upload the outer extracted folder.

UPLOAD

2. Select Add file > Upload files.
3. Drag the CONTENTS of the extracted folder.
4. Preserve all repository-relative directories.
5. Confirm no existing production file is shown as modified or deleted.
6. Confirm .github/workflows/release-232-governance.yml is included.
7. Confirm manifest-release-232.json is included.

- Do not create a second fix commit merely to add a note; the first-parent release identity must remain deterministic.
- If file coverage fails, compare Git changed files with manifest added_files.
- If checksum validation fails, stop and do not edit the ledger manually without regenerating the package.
- If reverse patch validation fails, do not continue to Release 233.
