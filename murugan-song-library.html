Correct information architecture

Collection → Work → Song → Reading formats

Individual repository files are not placed directly on the homepage. They become discoverable through a controlled hierarchy: library hub, collection page, song detail page, search index, source record and—where appropriate—audio.

No deletion. No false completeness.

Published full texts, reviewed extracts, navigation records and source registers remain visibly separated. A source-register card does not claim that a complete song text is available.

Standard song-detail format

Two Tamil reading forms, then meaning and audio

  1. Original Tamil
    Preserve the verified edition, metre and source wording.
  2. Easy-reading Tamil
    Provide carefully reviewed word spacing without silently changing the devotional text.
  3. Transliteration
    Add a consistent Roman transliteration for learners.
  4. Meaning
    Separate literal meaning, devotional explanation and copyrighted commentary.
  5. Audio
    Attach owned, licensed or clearly permitted recordings; otherwise use labelled device read-aloud.
  6. Source and status
    Display author, edition, source, completeness and review state on every song.

Repository recovery

Every song route is connected through governed discovery.

The full repository reachability programme separates genuine public devotional records from release evidence, checksums, test fixtures, generated scaffolding and mention-only files. Canonical content is promoted through route registry, search, sitemap and PWA integration; internal evidence remains preserved without cluttering devotee navigation.
